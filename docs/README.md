# Docs

팀 협업 시 참고할 문서 모음입니다.

## 문서 목록

- [CONVENTION.md](./CONVENTION.md): 작업 전반에서 지킬 기본 규칙
- [BRANCH.md](./BRANCH.md): 브랜치 생성 및 운영 규칙
- [COMMIT.md](./COMMIT.md): 커밋 메시지 규칙
- [STRUCTURE.md](./STRUCTURE.md): 현재 프로젝트 폴더 구조 설명
- [TROUBLESHOOTING.md](./TROUBLESHOOTING.md): 자주 만나는 문제와 해결 방법

## 추천 확인 순서

1. `CONVENTION.md`
2. `BRANCH.md`
3. `COMMIT.md`
4. `STRUCTURE.md`
5. `TROUBLESHOOTING.md`

## 빠른 시작

```bash
pnpm install
pnpm dev
```

## 문서 작성 원칙

- 문서는 팀 합의가 반영된 내용만 기록합니다.
- 임시 규칙은 문서에 반영하지 않습니다.
- 구조나 규칙이 변경되면 관련 문서를 함께 수정합니다.
- 작업 전에는 새 브랜치를 만들고, 커밋 메시지는 반드시 `COMMIT.md` 규칙을 따릅니다.
